# バリュー・プロポジション・キャンバス

## プロダクト名: ソーシャルギフトアセット

---

## 顧客プロファイル

### Customer Jobs（顧客のタスク）

#### 贈り手（企業・個人）
1. SNSキャンペーンの当選者にギフトを配布したい
2. 顧客へのお礼・特典をデジタルで届けたい
3. 配布したギフトの利用状況を把握したい

#### 受け取り手（エンドユーザー）
1. 受け取ったギフトを簡単に確認したい
2. 店舗でスムーズにギフトを利用したい
3. ギフトの有効期限や残高を管理したい

### Pains（痛み・不満）

1. **配送コストが高い**: 物理ギフトは送料・梱包費がかかる
2. **住所取得のハードル**: 個人情報を聞くのは心理的負担
3. **匿名性で追客できない**: 誰が受け取ったか分からず、CRMに活かせない
4. **受け取りが面倒**: 住所入力や再配達の手間
5. **チケット紛失リスク**: 紙のチケットは無くしやすい

### Gains（期待する成果）

1. 配送コストゼロでギフトを届けたい
2. 住所を聞かずにギフトを贈りたい
3. 受け取り手を顧客データ化したい
4. ワンタップでギフトを受け取りたい
5. スマホでいつでもチケットを確認したい

---

## バリューマップ

### Products & Services（提供するサービス）

1. **デジタルチケット発行**: URLベースのギフトチケット生成
2. **LINEシェア機能**: liff.shareTargetPicker()によるシームレスな共有
3. **会員化促進**: 受け取り時のLINEミニアプリ自動起動
4. **受け取り追跡ダッシュボード**: リアルタイムで受け取り状況を可視化
5. **QRコードチケット表示**: 店舗での利用をスムーズに

### Pain Relievers（痛みの解消方法）

| 痛み | 解消方法 |
|------|----------|
| 配送コストが高い | デジタル配送で物流コストをゼロ化 |
| 住所取得のハードル | URLシェアで住所不要に |
| 匿名性で追客できない | LINE連携で受け取り手を自動で顧客データ化 |
| 受け取りが面倒 | URLタップ→即座にチケット表示 |
| チケット紛失リスク | スマホに常にチケットを保存 |

### Gain Creators（成果の実現方法）

| 期待する成果 | 実現方法 |
|--------------|----------|
| 配送コストゼロ | 完全デジタル化でURL配信のみ |
| 住所不要でギフト | LINEメッセージでURL共有 |
| 顧客データ化 | LINEミニアプリ起動で自動会員登録 |
| ワンタップ受け取り | URLクリック→即チケット発行 |
| いつでも確認 | LINEミニアプリ内でチケット一覧表示 |
