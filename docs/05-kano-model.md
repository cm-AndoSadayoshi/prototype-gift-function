# 狩野モデル（機能優先順位）

## プロダクト名: ソーシャルギフトアセット

---

## 品質分類

### 当たり前品質 (Must-be) - 必須

ないと不満だが、あっても満足度は上がらない機能。

| 機能 | 説明 | MVP判断 |
|------|------|---------|
| ギフト商品一覧表示 | 選択可能なギフト商品を一覧表示 | ✅ 必須 |
| ギフトURL発行 | ユニークなギフトURLをリアルタイム生成 | ✅ 必須 |
| ギフト受け取り画面 | 受け取り手がギフト内容を確認できる | ✅ 必須 |
| デジタルチケット表示 | QRコード付きチケットを表示 | ✅ 必須 |
| 利用完了表示 | 店舗利用後の完了状態を表示 | ✅ 必須 |

---

### 一元的品質 (Performance) - 選択的

あればあるほど満足度が上がる機能。

| 機能 | 説明 | MVP判断 |
|------|------|---------|
| ギフト履歴一覧 | 過去に受け取ったギフトを一覧表示 | ⚪ 検討 |
| 受け取り追跡ダッシュボード | 管理者向けの配布・受け取り状況可視化 | ⚪ 検討 |
| 有効期限アラート | チケット期限前のプッシュ通知 | ⚪ 後回し |
| 複数商品ギフト | 複数商品をまとめて贈る | ⚪ 後回し |

---

### 魅力的品質 (Delighter) - 1つに集中

なくても不満はないが、あると感動する機能。

| 機能 | 説明 | MVP判断 |
|------|------|---------|
| **LINEシェア（shareTargetPicker）** | ワンタップでLINE友だちにシェア、ネイティブ体験 | ⭐ MVP採用 |
| 開封アニメーション | ギフト受け取り時の演出効果 | 後回し |
| SNSシェア機能 | Twitter/Instagram等へのシェア | 後回し |

---

## MVP機能サマリー

### 採用機能

| 品質タイプ | 機能 |
|------------|------|
| 当たり前品質 | ギフト商品一覧、URL発行、受け取り画面、チケット表示、利用完了 |
| 魅力的品質（1つ） | **LINEシェア（liff.shareTargetPicker）** |

### 見送り機能

| 機能 | 理由 |
|------|------|
| ギフト履歴一覧 | コアフローには不要、後から追加可能 |
| 受け取り追跡ダッシュボード | 管理機能は別フェーズで実装 |
| 有効期限アラート | プッシュ通知の実装コストが高い |
| 開封アニメーション | デモでは省略、本番で検討 |

---

## 魅力的品質の選定理由

### LINEシェア（liff.shareTargetPicker）を選んだ理由

1. **住所不要のコアバリューを体現**: URLをLINEで直接送れることが最大の差別化
2. **ネイティブ体験**: LINEの友だち選択UIがそのまま使え、ユーザーに馴染みがある
3. **実装コスト対効果**: LIFF SDKで比較的容易に実装可能
4. **デモ映え**: 営業デモでインパクトを与えやすい

### プロトタイプでの再現方法

- 実際のLIFF APIは使用せず、モックUIで再現
- 友だち選択画面のモックを表示
- 送信完了のフィードバックをシミュレーション
